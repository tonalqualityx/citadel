## Dashboard Philosophy (ND-Optimized)

These principles guide all dashboard design decisions:

| Principle | Dashboard Application |
|-----------|----------------------|
| **"What should I do next?"** | Surface ONE recommended action, not a wall of options |
| **TODAY by default** | Show today's work; require explicit action to see more |
| **3-5 items max** | Each section shows limited items with "View all" escape hatch |
| **Context preservation** | Recent items sidebar helps recover from interruptions |
| **No anxiety triggers** | No red "overdue" shame; neutral language for alerts |
| **Progressive disclosure** | Collapsed sections for less-urgent information |

### Tech Overlook Sections

| Section | Purpose | Default State | Max Items |
|---------|---------|---------------|-----------|
| **Active Timer** | Current work context | Visible (or "no timer" state) | 1 |
| **What's Next?** | Single recommended task | Visible, prominent | 1 |
| **My Quests (Ready)** | Tasks ready to work | Visible | 5 |
| **Blocked/Upcoming** | Not actionable now | Collapsed | 5 |
| **Missing Time** | Gaps in time tracking | Visible only if gaps exist | 7 days |
| **Recent Time** | Today/yesterday entries | Visible, summarized | 2 days |

### "What's Next?" Logic

The recommended task is selected by this priority order:

1. **Highest priority** among Ready tasks assigned to user
2. If tie â†’ **Earliest due date**
3. If tie â†’ **Lowest energy** (quick win)
4. User can "skip" to see full quest list

## Summary: ND Principles Applied to Dashboards

| Principle | Implementation |
|-----------|----------------|
| **"What's next?"** | Single recommended task for Techs, reducing decision paralysis |
| **TODAY by default** | Shows today's work and time; historical data requires action |
| **3-5 items max** | Each section limited; "View all" for more |
| **No anxiety triggers** | "Missing time" not "OVERDUE!"; neutral amber for alerts, no red |
| **Context preservation** | Recent items sidebar; breadcrumbs in all cards |
| **Progressive disclosure** | Blocked/upcoming and secondary sections collapsed by default |
| **Inline actions** | Review approve/return without leaving dashboard |
| **Clear visual grouping** | Cards with borders; whitespace between sections |
| **Predictable layout** | Same section order every time; no dynamic reordering |
| **Explicit states** | Clear empty states with helpful messaging |
