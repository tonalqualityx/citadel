## Overview

This document defines the wireframes for Indelible's global shellâ€”the persistent UI wrapper that contains navigation, header, and content areas across all authenticated views. All design decisions incorporate neurodivergent optimization principles as foundational requirements, not afterthoughts.

---

## Neurodivergent Design Principles Applied

These principles drive every decision in the shell design:

| Principle | Shell Application |
|-----------|-------------------|
| **Predictability** | Navigation stays in identical positions across ALL views |
| **Reduce cognitive load** | 3-5 visible options max before progressive disclosure kicks in |
| **Context preservation** | Breadcrumbs + "where was I?" indicators always visible |
| **Visual comfort** | Warm off-white / dark gray palette, no harsh contrasts |
| **Icons + labels** | ALWAYS pair icons with text (no unlabeled iconography) |
| **Explicit over implicit** | States, actions, and feedback are crystal clear |
| **Respect motion preferences** | CSS `prefers-reduced-motion`, no surprise animations |

## Animation Guidelines

Following ND research: default to no animation, enable only for users who prefer it.

### Default Behavior

- Respect `prefers-reduced-motion` media query
- No animation on page transitions
- No animation on sidebar expand/collapse
- No animation on dropdown open/close

### Allowed Animations (When Motion OK)

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Toast appear | Fade in | 150ms | ease-out |
| Toast dismiss | Fade out | 100ms | ease-in |
| Focus ring | None (instant) | 0ms | â€” |
| Hover states | Background color | 100ms | ease |
| Loading spinner | Rotate | 1000ms | linear |

### Never Animate

- Zoom effects
- Parallax scrolling
- Bounce effects
- Slide-in navigation
- Auto-playing carousels

## Accessibility Checklist

| Requirement | Implementation |
|-------------|----------------|
| **Focus visible** | 2px accent color ring on all interactive elements |
| **Color contrast** | Minimum 4.5:1 for text, 3:1 for UI elements |
| **Skip links** | "Skip to main content" link on focus |
| **ARIA labels** | All icon-only buttons have aria-label |
| **Keyboard navigation** | All functions accessible via keyboard |
| **Screen reader** | Semantic HTML, proper heading hierarchy |
| **Motion** | Respect prefers-reduced-motion |
| **Text resize** | Layout works at 200% zoom |

## Summary: ND Principles Applied

| ND Principle | How It's Applied in Shell |
|--------------|---------------------------|
| **Predictability** | Sidebar and header positions never change; consistent across all views |
| **Progressive disclosure** | Sidebar sections collapse; search shows recent first, then results |
| **Context preservation** | Breadcrumbs always visible; timer shows current task context |
| **3-5 options max** | Top-level nav has 5 items; sections expand on demand |
| **Icons + labels** | Every nav item and action has both |
| **Visual comfort** | Warm off-white, no pure white/black, muted accent colors |
| **No surprise animation** | Respects `prefers-reduced-motion`; only subtle fades |
| **Explicit feedback** | Toasts confirm actions; states are clearly labeled |
| **Clear boundaries** | Cards, sections, and whitespace create visual grouping |
| **Persistent labels** | Form fields use labels, never placeholder-only |
